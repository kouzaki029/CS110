Forms can be defined using the <form> tag, <label> tag, <input> tag, <option> tag, <button> tag, <textarea> tag, and the <fieldset> tag.


<form>
   <label for="name">Name:</label>
   <input type="text" id="name" name="name" required>

   <label for="email">E-mail:</label>
   <input type="email" id="email" name="email">

   <label for="programmingLanguage">Favorite Programming Language:</label>
   <select id="programmingLanguage" name="programmingLanguage">
      <option value="javascript">Javascript</option>
      <option value="python">Python</option>
      <option value="java">Java</option>
      <option value="csharp">C#</option>
   </select>
   <button type="submit">Submit:</button>
</form>

To create a form, you need to first include the <form> tag, followed by an ending </form> tag to indicate where the last line of the form is.

It is important to note, similar to the <form> tag, the <label> tag, the <select> tag, the <option> tag, the <button> tag, the <textarea> tag, and the <fieldset> tag all must have an ending tag. In essence, </form> , </label> , </select> , </option> , </button> , </textarea> , </fieldset> respectively.

Within the <form> tags, you can include which ever form element you want (i.e., <label> tag, <input> tag, <select> tag, <button> tag, <textarea> tag, or even the <fieldset> tag), however it is best practice to use the <label> tag first to indicate to the user the purpose of the following element is used for (e.g., "Participant's Name:" is before the textbox in Figure 1).

The <input> tag, supports various form components, such as radio buttons, checkboxes, submit buttons, buttons, and textboxes.

To create a radio button, you write a <input type="radio"> tag.

To create a checkbox, you write a <input type="checkbox"> tag.

To create a submit button, you write a <input type="submit"> tag.

To create a button, you write a <input type="button"> tag.

To create a textbox, you write a <input type="text"> tag.

The name and id attributes are used to identify a HTML element.

Unlike name, id attribute must be unique - no two HTML elements can share the same id otherwise the two elements become ambiguous. A good example could be if there are 25 students in a class but there are three students with the name "John", even though they all share the same name, each John has a unique student id that distinguishes them. If one John has an overdue book, then one of the other Johns won't be fined because each John has a unique.

Buttons can also be created using the <button> tag. Two types the <button> tag can have are "button" and "submit".

The <select> tag is used to have a drop-down selection for the user. To create items in the drop-down selection, the <option> tag is used.

The <textarea> tag is used to have a multiple line textbox. The <textarea> tag is commonly used for paragraph responses on forms.

The <fieldset> tag is used to group common HTML elements (for example a form need to collect the user's name, address, phone, etc, but all of these can be group as "Contact Info"). The <fieldset> tag can also be used to draw emphasis to the form.
